# Изолированный локальный preview

Этот режим запускает полнофункциональный пользовательский интерфейс «Путь» без
production-секретов, Telegram и общей Docker-базы. Он предназначен только для
локального просмотра и browser QA.

## Запуск одной командой

Требуются установленные зависимости API и web. Из корня репозитория:

```powershell
python scripts/run_test_preview.py --reset
```

После готовности runner сам откроет `http://127.0.0.1:55173/app` в браузере. Первый запуск начинает
онбординг, последующие сохраняют состояние в `.test-preview/preview.db`. Без
`--reset` можно продолжить тот же путь; с `--reset` удаляется только эта
изолированная preview-БД.

Для автоматизированного запуска без открытия браузера добавьте `--no-open`.
Если API был перезапущен, открытая вкладка один раз безопасно обновит только
development-сеанс и продолжит с сохранённого экрана; production-вход это не меняет.

Если preview уже работает, повторный запуск без `--reset` просто напечатает его
адрес. Для сброса сначала остановите активный процесс через `Ctrl+C`: runner не
удаляет открытую БД. Если один из preview-портов занят другой программой, запуск
останавливается с указанием конкретного порта.

Скрипт:

- применяет актуальные Alembic-миграции;
- поднимает API и Vite только на loopback-интерфейсе;
- создаёт отдельный development-only bearer-сеанс;
- не читает и не изменяет обычную Docker/PostgreSQL-базу;
- завершает оба процесса по `Ctrl+C`.

Маршрут `/v1/auth/development` требует явного
`DEVELOPMENT_AUTH_ENABLED=true`, работает только при
`APP_ENVIRONMENT=development` и запрещён production-конфигурацией. Этот режим
нельзя использовать для staging, production, пилота или реального UAT.
