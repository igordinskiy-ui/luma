# ADR-001: аутентификация пользователя

## Решение

API принимает Telegram `initData` только на `/v1/auth/telegram`, проверяет подпись серверным токеном бота и выдаёт подписанную краткоживущую bearer-сессию. Все пользовательские API работают только от этой сессии; Telegram ID не передаётся в URL и не принимается как идентификатор доступа.

## Последствия

Для локальной разработки нужен тестовый подписанный payload либо отдельный dev-auth adapter, не включаемый в production. Полноценный Telegram Login/OIDC для автономного браузерного PWA добавляется следующим изменением после регистрации OAuth-клиента в BotFather.
