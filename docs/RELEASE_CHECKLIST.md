# Чек-лист закрытой и открытой B2C-беты

Каждая отметка требует ссылки на доказательство: ticket/PR, имя ответственного,
дату, скриншот или журнал команды. «Настроено» без такого доказательства не
является закрытым gate. Для публичного запуска все пункты раздела «Блокеры»
должны быть закрыты владельцем продукта и ответственным за безопасность.

## Блокеры до приёма публичных пользователей

Закрытый инфраструктурный preview можно развернуть до закрытия этих пунктов
только с `PUBLIC_LAUNCH_ENABLED=false`. В этом режиме вход, пользовательский API
и доставка уведомлений должны оставаться заблокированными.

- [ ] **Исполнимый preflight.** На production-хосте с фактическими secret-manager
  значениями выполнен `python scripts/release_preflight.py`; сохранён только
  результат команды без секретов.

- [ ] **Оператор и документы.** Holding pages заменены утверждёнными `privacy.html` и `terms.html`; юрист подтвердил актуальность реквизитов, оснований обработки, сроков, исполнителей, локализации/трансграничной передачи и порядка споров. `LEGAL_DOCUMENTS_VERSION` уникальна, а `LEGAL_DOCUMENTS_DIGEST` совпадает с `python scripts/legal_manifest.py`. Доказательство: ссылка на утверждённую редакцию, дата, version и digest.
- [ ] **152-ФЗ: реестр и роли.** Владелец обработки, ответственный, перечень ИСПДн, категории субъектов/данных, цели, правовые основания, сроки, доступы и подрядчики внесены в внутренний реестр. Проверено, требуется ли уведомление Роскомнадзора и выполнено требуемое действие. Доказательство: номер записи/решения юриста.
- [ ] **Согласие и возраст.** Юрист подтвердил текст, способ фиксации, версию и доказательство согласия; определён возрастной порог и сценарий для несовершеннолетних либо ограничение доступа. Доказательство: UAT первичного и повторного согласия, неизменяемая запись `version + digest + accepted_at` в тестовом экспорте и сохранение прежней записи после re-consent.
- [ ] **Данные повышенной чувствительности.** Проведена квалификация заметок и записей о привычках/самочувствии; если применимо, отдельно утверждены основания и форма согласия для специальных категорий. Доказательство: заключение юриста/ответственного.
- [ ] **Подрядчики и хостинг.** Подписаны поручения/договоры с хостингом, мониторингом, почтой, push и иными обработчиками; подтверждены регион первичной записи данных РФ, субподрядчики, трансграничность и порядок уведомления. Доказательство: реестр обработчиков и договорные ссылки.
- [ ] **Retention и удаление.** Утверждена таблица сроков для активной БД, delivery/analytics/support-логов и резервных копий; проверены экспорт, удаление и невозможность вернуть удалённые данные при штатном restore после срока ротации. Доказательство: тест-кейс с ID тестового пользователя и результатом restore drill.
- [ ] **Реагирование на инциденты.** Назначены 24/7 контакты, severity/SLA, журнал инцидентов и юридический маршрут оценки/уведомления. Проведено tabletop-упражнение утечки. Доказательство: датированный протокол и action items.
- [ ] **Контент.** Медредактор утвердил материалы по `CONTENT_REVIEW_CHECKLIST.md`; `CONTENT_REVIEW_STATUS=approved`, полный `CONTENT_APPROVED_DIGEST` и runtime `CONTENT_CATALOGUE_DIGEST` совпадают с committed copy и установлены только в production secret. Доказательство: дата/подпись review, оба digest и deploy record.
- [ ] **Production-инфраструктура.** Установлены production `SESSION_SECRET`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, HTTPS, домен, Telegram Web App URL и webhook. Доказательство: secret-manager references, `/health` и `getWebhookInfo` без ошибок.
- [ ] **Наблюдаемость.** Настроены error tracking и панели/алерты по `OBSERVABILITY_RUNBOOK.md`: API p95/error rate, worker heartbeat, queue/database/delivery/backup; назначены владельцы и выполнен тест доставки. Доказательство: ссылки на dashboards и alert test.
- [ ] **Backup/restore.** Резервные копии зашифрованы на уровне хранилища, имеют ограниченный доступ и retention; проведено восстановление в изолированную среду, проверены целостность, RPO/RTO и удаление тестовой среды. Доказательство: отчёт restore drill.
- [ ] **Безопасность релиза.** Все remote Actions закреплены full commit SHA; пройдены актуальные dependency/secret/container scans, рассмотрены результаты security-аудита, утверждены исключения. Доказательство: CI run после pinning и risk acceptance.
- [ ] **UAT.** На deployed host завершены Android/iOS Telegram Mini App, PWA и desktop сценарии: login, consent, quit flow, уведомление, feedback, export, deletion. Доказательство: UAT-таблица с устройствами и дефектами.

## Контролируемая закрытая бета

- [ ] Определены лимит пользователей, критерии отбора, владелец поддержки, часы ответа и канал для privacy/medical escalation.
- [ ] Еженедельно рассматриваются activation, D1/D7/D14, mute rate, delivery failures, обращения, инциденты и расходы; решение и действия зафиксированы.
- [ ] Изменения risk-score выпускаются только за feature flag с rollback-планом.
- [ ] Все privacy-запросы получают ID, подтверждение личности, срок, решение и доказательство выполнения; обращения не хранятся дольше утверждённого срока.
- [ ] Не публиковать рекламные заявления о лечении, гарантированном результате или продвижении табачных товаров; все креативы проходят legal/content review.

## Go/no-go после минимум 4 недель пилота

- [ ] Владелец продукта подписал итог: retention и activation, безопасность/инциденты, качество поддержки, жалобы, delivery cost и технический долг.
- [ ] Нет открытых критических инцидентов, просроченных privacy requests или неустранённых high-risk юридических замечаний.
- [ ] Решена следующая фаза (публичная бета, платежи, новые категории или B2B2C), и для неё создан отдельный legal/security review.
